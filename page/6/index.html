1. 表分区说明

在db2数据库中，可对数据表按某个字段进行分区，分区好处是：可扩展表的存储能力、对于大量数据表使用分区字段可提高查询效率。表分区不同于数据库分区，需要在创建表的时候设置分区逻辑。

　 在定义表分区的字段时，有几个原则：1.不支持创建只包含长数据类型的多分区表，2.不能改变分区键定义，3.分区键应该包含最频繁连接的列，4.分区键应该由经常参与group by字句的组成，5.任何主键或唯一键必须包含分区列

DB2

一、分区表优势

  1. 数据移动：

    周期性地添加大量数据，然后在数据到期后又删除大量数据；通常每月或每个季度删除过时的大量数据的表。

  2. 表分区通过分区排除方式提高查询性能：

    假设 Transactions 历史表有 36 个分区，每个月对应一个分区。对于一个选择过去 12 个月的数据的查询，优化器知道不必扫描这 12 个月之外的其他分区的数据。这种分区排除既适用于索引扫描，也适用于表扫描。

    表分区只对那些包括表分区键列的查询有利。

二、常规表转变为分区表方案

1. 新建缓冲池

2. 为分区创建独立表空间

3. 创建分区表

4. 迁移数据

5. 验证无误后删除旧表，新表改名

DB2创建索引

1、创建索引

create index 索引名 on 表名（列名，列名）；

2、删除索引

drop index 索引名 on 表名；

3、查看表索引

select * from sysibm.sysindexes where tbname = ‘表名’；–表名要区分大小写

或者数据库后台可以用

describe indexes for table 表名;

4、查看SQL语句执行计划

db2expln -d 库名 -f test.sql -t -g -z “;” > test.exp

5、查看SQL语句索引优化建议

db2advis -d 库名 -i test.sql
